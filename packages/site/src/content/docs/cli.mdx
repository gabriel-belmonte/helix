# CLI reference

The `helix` CLI is the primary way to run the agent. Install it from the
[releases page](https://github.com/gabriel-belmonte/helix/releases) or with:

```bash
curl -fsSL https://raw.githubusercontent.com/gabriel-belmonte/helix/main/packages/cli/install.sh | sh
```

## Run a prompt

```bash
helix -p "refactor utils.ts to async/await"   # single shot, then exit
helix                                          # interactive REPL
helix -v -p "list files in src/"              # verbose: show tool calls
```

| Flag | Meaning |
|------|---------|
| `-p, --prompt <text>` | run one prompt and exit |
| `-v, --verbose` | print tool calls as they happen |
| `-c, --cwd <dir>` | run with a different working directory |
| `--scripted` | use the built-in deterministic LLM (no network, no keys) — great for testing |

## Config

```bash
helix config set provider zen        # one of: zen | hf | openrouter | openai
helix config set model big-pickle
helix config get                     # show full config
helix config get model               # show one key
helix config list                    # show config path + values
```

**Fallback providers** (auto-failover if the first model fails):

```bash
# Set multiple fallbacks via the `fallback` config key:
helix config set fallback '["zen:mimo-v2.5-free","hf:Qwen/Qwen3-Coder-Next"]'

# Or edit ~/.helix/config.json directly:
#   { "provider": "zen", "model": "big-pickle",
#     "fallback": ["zen:mimo-v2.5-free", "hf:Qwen/Qwen3-Coder-Next"] }
```

When `fallback` is set, Helix tries each entry in order. If the first model
errors, it moves to the next — so you always have a working model. See
[`helix status`](#status) for the active chain.

## Compression

Helix ships two optional compression plugins that save tokens on tool output:

| Plugin | What it does | Savings | Enable in config |
|--------|-------------|---------|-----------------|
| **Caveman** | Drops articles, shortens words, removes filler. Ultra-compact speak. | 40–60% | `"features": {"caveman": true}` |
| **RTK** | Truncates long lines, condenses lists to head+tail, collapses whitespace. | 30–70% | `"features": {"rtk": true}` |

```bash
# Enable both in ~/.helix/config.json:
# { "features": { "caveman": true, "rtk": true } }
```

They stack: RTK runs first (structural compression), then Caveman (textual
compression). Together they can reduce tool output tokens by **50–80%** with
minimal information loss.

## Web search

Helix can search the web with two tools: `web_search` (search engine) and
`web_extract` (page content). Enable them per-command with the `--web` flag:

```bash
helix --web -p "search for latest TypeScript news"
helix --web-search -p "what is the weather in London?"  # search only
helix --web-extract -p "summarize https://example.com"   # extract only
```

To keep web enabled **by default** (no `--web` flag needed), add it to config:

```bash
# Via CLI:
helix config set web '{"search":true,"extract":true}'

# Or edit ~/.helix/config.json:
#   { "web": { "search": true, "extract": true } }
```

CLI flags (`--web`, `--web-search`, `--web-extract`) always take precedence
over the stored config — use them to temporarily override.

Web tools need a running SearXNG instance (or other search backend). The
containerized Helix Dashboard ships with one; standalone users can point
Helix at their own via `infra` config.

## API keys (auth)

```bash
helix auth login zen                 # prompt for a key (hidden input), saved to ~/.helix/auth.json
helix auth list                      # show every provider + whether its key is set (masked)
helix auth logout zen                # delete the stored key
```

Keys can also come from environment variables (`OPENCODE_ZEN_API_KEY`,
`HF_TOKEN`, `OPENROUTER_API_KEY`, `OPENAI_API_KEY`). **Env vars always win**
over stored keys.

## Models

```bash
helix models            # list the OpenCode Zen catalog (free models highlighted)
helix models select     # interactive picker → saves to config
helix models set <id>   # set the model directly
```

## Status

```bash
helix status
```

Prints the **active provider + model** and the **API-key status of every
provider**. See also [`helix doctor`](#doctor) for a full diagnostics check.

## Doctor

```bash
helix doctor
```

Runs a full diagnostics check:

- **API keys** — which providers have keys configured (env or stored)
- **SearXNG (web_search)** — checks if the search backend is online on :8888
- **Extract (web_extract)** — checks if the extract server is online on :8787
- **Docker** — whether Docker is available on the system
- **Skills** — how many SKILL.md folders are discovered
- **Config** — provider and model status

Useful when something isn't working — tells you exactly what's misconfigured.

## Init

```bash
helix init
```

Interactive setup wizard. If no provider is configured yet, it sets up
`~/.helix/config.json` with sensible defaults (provider=zen, model=big-pickle)
and prints next steps for setting your API key, trying the TUI, and launching
the Dashboard.

## Session

```bash
helix session list                      # show saved conversations
helix session save my-analysis          # save current conversation
helix session load my-analysis          # restore a saved conversation
helix session export my-analysis        # print session as JSON
```

Conversations are stored in `~/.helix/sessions/` as JSON files. Useful for
bookmarking progress, sharing context, or debugging what the model saw.

## JSON output

```bash
helix --json -p "analyze this code"
```

Returns structured JSON instead of plain text, useful for scripting / CI:

```json
{
  "result": "…analysis…",
  "latency": 3200,
  "turns": 1
}
```

## Evaluate (eval)

```bash
helix eval --suite suite.json                       # run one model over a suite
helix eval --suite suite.json --model deepseek-v4-flash-free
helix eval --suite suite.json --compare north-mini-code-free   # A/B two models
helix eval --suite suite.json --judge deepseek-v4-flash-free    # grade with an LLM judge
```

See [Model evaluation](/helix/eval/) for the suite format and worked examples.

## Version

```bash
helix --version     # or: helix -V
# → helix-agent-cli 0.1.0
```

## Dashboard

```bash
helix dashboard    # launch the web Dashboard on http://localhost:8799
```

Opens the control panel (config, API keys, skills, MCP, files). With no key set
it starts in demo mode. See [Dashboard (Web)](/helix/web-dashboard/) for the
full guide, including how to paste your API key from the browser.

## TUI

```bash
helix tui    # launch the terminal UI (Ink-based chat)
```

Launches a full-screen chat interface with streaming output, a live tool-call
indicator, and a model picker (Ctrl+M). Needs a TTY.

## Update

```bash
helix update        # re-download the latest release binary
```
