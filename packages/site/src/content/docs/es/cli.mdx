# Referencia de la CLI

La CLI `helix` es la forma principal de ejecutar el agente. Instálala desde la
[página de releases](https://github.com/gabriel-belmonte/helix/releases) o con:

```bash
curl -fsSL https://raw.githubusercontent.com/gabriel-belmonte/helix/main/packages/cli/install.sh | sh
```

## Ejecutar un prompt

```bash
helix -p "refactoriza utils.ts a async/await"   # un disparo y sale
helix                                          # REPL interactivo
helix -v -p "lista los archivos en src/"        # verbose: muestra las llamadas a tools
```

| Flag | Significado |
|------|-------------|
| `-p, --prompt <texto>` | ejecuta un prompt y sale |
| `-v, --verbose` | imprime las llamadas a tools en vivo |
| `-c, --cwd <dir>` | ejecuta en otro directorio de trabajo |
| `--scripted` | usa el LLM determinista integrado (sin red, sin claves) — ideal para probar |
| `--sandbox` | ejecuta dentro de un contenedor Docker sandbox (implica `-p`) |
| `--json` | output JSON estructurado para scripting / CI |
| `--web` | activa `web_search` (SearXNG) + `web_extract` (compatible Firecrawl) |
| `--web-search` | activa solo búsqueda web |
| `--web-extract` | activa solo extracción web |

## Config

```bash
helix config set provider zen        # uno de: zen | hf | openrouter | openai | anthropic
helix config set model big-pickle
helix config get                     # muestra toda la config
helix config get model               # muestra una clave
helix config list                    # muestra la ruta de config + valores
```

**Fallback de proveedores** (failover automático si el primer modelo falla):

```bash
# Poner múltiples fallbacks con la clave `fallback`:
helix config set fallback '["zen:mimo-v2.5-free","hf:Qwen/Qwen3-Coder-Next"]'

# O editar ~/.helix/config.json directamente:
#   { "provider": "zen", "model": "big-pickle",
#     "fallback": ["zen:mimo-v2.5-free", "hf:Qwen/Qwen3-Coder-Next"] }
```

Cuando `fallback` está configurado, Helix prueba cada entrada en orden. Si
el primer modelo da error, salta al siguiente — así siempre tienes un modelo
funcionando. Ve [`helix status`](#estado-status) para la cadena activa.

## Compresión

Helix incluye dos plugins opcionales de compresión que ahorran tokens en el
output de herramientas:

| Plugin | Qué hace | Ahorro | Activar en config |
|--------|----------|--------|-------------------|
| **Caveman** | Elimina artículos, acorta palabras, quita relleno. Habla ultra-compacta. | 40–60% | `"features": {"caveman": true}` |
| **RTK** | Trunca líneas largas, condensa listas a cabeza+cola, colapsa espacios. | 30–70% | `"features": {"rtk": true}` |

```bash
# Activar ambos en ~/.helix/config.json:
# { "features": { "caveman": true, "rtk": true } }
```

Se apilan: RTK primero (compresión estructural), luego Caveman (compresión
textual). Juntos reducen los tokens de output de herramientas un **50–80%**
con pérdida mínima de información.

## Búsqueda web

Helix puede buscar en la web con dos tools: `web_search` (motor de búsqueda) y
`web_extract` (contenido de página). Actívalas por comando con el flag `--web`:

```bash
helix --web -p "busca las últimas noticias de TypeScript"
helix --web-search -p "¿qué tiempo hace en Londres?"    # solo búsqueda
helix --web-extract -p "resume https://example.com"      # solo extracción
```

Para mantener web activo **por defecto** (sin flag `--web`), añádelo a la config:

```bash
# Vía CLI:
helix config set web '{"search":true,"extract":true}'

# O editar ~/.helix/config.json directamente:
#   { "web": { "search": true, "extract": true } }
```

Los flags CLI (`--web`, `--web-search`, `--web-extract`) siempre tienen prioridad
sobre la config guardada — úsalos para sobreescribir temporalmente.

Las herramientas web necesitan una instancia de SearXNG ejecutándose (u otro
backend de búsqueda). El Dashboard containerizado de Helix incluye una; los
usuarios standalone pueden apuntar Helix a la suya vía la config `infra`.

## Claves API (auth)

```bash
helix auth login zen                 # pide una clave (oculta), la guarda en ~/.helix/auth.json
helix auth list                      # muestra cada proveedor y si su clave está (enmascarada)
helix auth logout zen                # borra la clave guardada
```

También puedes usar variables de entorno (`OPENCODE_ZEN_API_KEY`, `HF_TOKEN`,
`OPENROUTER_API_KEY`, `OPENAI_API_KEY`). **Las env vars siempre ganan** sobre
las claves guardadas.

## Modelos

```bash
helix models            # lista el catálogo de OpenCode Zen (modelos gratis resaltados)
helix models select     # selector interactivo → guarda en config
helix models set <id>   # fija el modelo directamente
```

## Estado (status)

```bash
helix status
```

Muestra el **provider + modelo activos** y el **estado de la API key de cada
provider**. Ve también [`helix doctor`](#doctor) para un diagnóstico completo.

## Doctor

```bash
helix doctor
```

Ejecuta un diagnóstico completo:

- **API keys** — qué proveedores tienen clave configurada (env o almacenada)
- **SearXNG (web_search)** — comprueba si el backend de búsqueda está online en :8888
- **Extract (web_extract)** — comprueba si el servidor de extracción está online en :8787
- **Docker** — si Docker está disponible en el sistema
- **Skills** — cuántos directorios SKILL.md se encontraron
- **Config** — estado del proveedor y modelo

Útil cuando algo no funciona — te dice exactamente qué está mal configurado.

## Init

```bash
helix init
```

Asistente de configuración interactivo. Si no hay proveedor configurado,
establece valores por defecto (provider=zen, model=big-pickle) y muestra
los siguientes pasos.

## Sesión

```bash
helix session list                      # muestra conversaciones guardadas
helix session save mi-analisis          # guarda la conversación actual
helix session load mi-analisis          # restaura una conversación guardada
helix session export mi-analisis        # imprime la sesión como JSON
```

Las conversaciones se guardan en `~/.helix/sessions/` como JSON. Útil para
marcar progreso, compartir contexto o depurar lo que vio el modelo.

## Salida JSON

```bash
helix --json -p "analiza este código"
```

Devuelve JSON estructurado en vez de texto plano, útil para scripting / CI:

```json
{
  "result": "…análisis…",
  "latency": 3200,
  "turns": 1
}
```

## Evaluar (eval)

```bash
helix eval --suite suite.json                       # ejecuta un modelo sobre una suite
helix eval --suite suite.json --model deepseek-v4-flash-free
helix eval --suite suite.json --compare north-mini-code-free   # A/B de dos modelos
helix eval --suite suite.json --judge deepseek-v4-flash-free    # puntúa con un LLM judge
```

Ve [Evaluación de modelos](/helix/es/eval/) para el formato de la suite y ejemplos.

## Versión

```bash
helix --version     # o: helix -V
# → helix-agent-cli 0.1.0
```

## Dashboard

```bash
helix dashboard    # lanza el Dashboard web en http://localhost:8799
```

Abre el panel de control (config, API keys, skills, MCP, archivos). Sin key
arranca en modo demo. Ve [Dashboard (Web)](/helix/es/web-dashboard/) para la
guía completa, incluyendo cómo pegar tu API key desde el navegador.

## TUI

```bash
helix tui    # lanza la interfaz de terminal (chat con Ink)
```

Abre una interfaz de chat a pantalla completa con output en streaming,
indicador de tool calls en vivo y selector de modelos (Ctrl+M). Necesita una TTY.

## Actualizar

```bash
helix update        # re-descarga el binario de la última release
```
