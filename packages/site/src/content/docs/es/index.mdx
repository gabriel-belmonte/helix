# Helix

**Helix es un framework de agentes TypeScript minimalista y transparente** — un
motor que puedes leer de verdad, más un ecosistema modular en crecimiento donde
cada capacidad (búsqueda, extracción, shell, archivos…) es un módulo
intercambiable.

Empezó como un motor de ~300 LOC + un CLI `curl | sh`. Desde entonces ha
crecido en un monorepo con una **fuente única de verdad** (`helix-core`) que
comparte cada superficie (CLI, futura TUI, web UI, Dashboard, Desktop) — y un
**sistema de plugins** para que puedas reemplazar o añadir cualquier módulo sin
hacer fork.

<CardGrid>

## Paquetes

| Paquete | Rol |
|---------|------|
| `helix-agent` | El motor puro: bucle del agente, dispatch de tools, parsers de formato. |
| `helix-core` | **Fuente única de verdad** — runtime, registro, plugins, config. |
| `helix-agent-cli` | Agente de terminal (`helix …`), binario nativo. |
| `helix-agent-eval` | Detección de regresiones de calidad / coste / latencia. |
| `helix-site` | Este sitio + docs (Astro + Starlight). |

## Filosofía

- **Legible antes que ingenioso.** Sin 40 clases abstractas, sin magia oculta.
- **Trae tu propio LLM.** Cualquier provider tras `complete(messages)` funciona.
- **Módulos que controlas.** Activa/desactiva funciones; cámbialas por las tuyas.
