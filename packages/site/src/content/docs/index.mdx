# Helix

**Helix is a minimal, transparent TypeScript agent framework** — an engine you
can actually read, plus a growing, **modular ecosystem** where every capability
(search, extract, shell, files…) is a swappable module.

It started as a ~300 LOC engine + a `curl | sh` CLI. It has since grown into a
monorepo with a **single source of truth** (`helix-core`) that every surface
(CLI, future TUI, web UI, Dashboard, Desktop) shares — and a **plugin system**
so you can replace or add any module without forking.

## Packages

| Package | Role |
|---------|------|
| `helix-agent` | The raw engine: agent loop, tool dispatch, format parsers. |
| `helix-core` | **Single source of truth** — runtime, registry, plugins, config. |
| `helix-agent-cli` | Terminal coding agent (`helix …`), native binary. |
| `helix-agent-eval` | Quality / cost / latency regression detection. |
| `helix-site` | This site + docs (Astro + Starlight). |

## Philosophy

- **Readable over clever.** No 40 abstract classes, no hidden magic.
- **Bring your own LLM.** Any provider behind `complete(messages)` works.
- **Modules you control.** Turn features on/off; swap them for your own.
